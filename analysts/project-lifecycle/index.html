Project lifecycle for Analysts

To become more familiar with the end-to-end workflow for data analysts in Prophecy, complete this tutorial!

Objectives​

In this tutorial, you will:

  • Create a Prophecy fabric and a new SQL project.
  • Develop a pipeline using various data sources and transformations.
  • Create a schedule to run the pipeline on a regular basis.
  • Save and publish the project.
  • Review your deployed projects and pipelines.

Requirements​

For this tutorial, you will need:

  • Prophecy 4.0 or later.
  • A Databricks or Snowflake SQL warehouse.

Create a fabric​

Before you can work on a project, you need to configure where that project will be computed. For this tutorial, you will use a Prophecy fabric to let you run pipelines in Prophecy Automate and your own SQL warehouse.

  1. Open Prophecy.
  2. Click on the Create Entity button in the left navigation bar.
  3. Select the Fabric tile.
  4. Give the fabric any name, like SQLPipelineTutorial.
  5. Select the team that will be able to use the fabric. For this tutorial, you might want to select your personal team. (It will match your individual user email.)
  6. Click Continue.

Next, you'll configure the Provider.

  1. Choose Prophecy as the Provider Type. This will enable the use of Prophecy Automate.
  2. Select your SQL provider (either Snowflake or Databricks).
  3. Click Continue.

In the Connections page, you'll add your SQL warehouse credentials.

  1. Click + Connect SQL Warehouse.
  2. Name the connection.
  3. Verify that the connection type matches your SQL provider.
  4. Enter the JDBC URL. It will look something like jdbc:databricks://<server-hostname>:443;httpPath=<http-path>[;<setting1>=<value1>;<settingN>=<valueN>].
  5. Enter the Catalog that will be used for target tables.
  6. Enter the Schema that will be used for target tables.
  7. Choose the appropriate Authentication method.
  8. Prophecy will automatically test the connection to validate it.
  9. Save the connection.

Create a project​

Now that you have a fabric, create a new project.

  1. Click on the Create Entity button in the left navigation bar.
  2. Hover over the Project tile and select Create.
  3. Give your project a name.
  4. Select the team that will have access to your project. Again, you might want to use your personal team.
  5. Select the Prophecy for Analysts template.
  6. Click Complete. This will open your new project.
  7. You will be prompted to select a fabric to continue. Choose the fabric that you created in the previous section.
  8. Click Save.

Build a pipeline​

In this step, you'll add your first pipeline to this project!

  1. Hover Pipelines in the left sidebar.
  2. Click on the plus sign.
  3. Name your pipeline.
  4. Keep the default directory path.
  5. Click Create.

Add source data​

At this point, the pipeline canvas is empty. Let's add some gems! To begin, you'll create a Seed as the data source.

  1. Open the Source/Target gem category.
  2. Click Table. This adds a new Table gem to the canvas.
  3. Hover over the gem and click Open.

Fill in the gem configuration.

  1. Select + New Table.
  2. For the Type and Format, choose Seed.
  3. Name the seed web_traffic.
  4. For the Seed path, choose seeds. This is the directory where the seed is saved in the backend.
  5. Click Next.

Transform the data​

Now, you'll configure your first data transformation using the Aggregate gem.

  1. From the Transform gem category, add a Aggregate gem to your canvas.
  2. Drag the Transform gem near your Table gem to auto-connect them.
  3. Open the Transform gem configuration.
  4. In the Group By Columns tile, add the page_url column as an expression.
  5. In the Expressions tile, add three target columns:
    • page_url. This column will contain the original page URL.
    • page_url_count. This column will contain the count of records in the table per URL.
    • page_average_session. This column will contain the average session time users spent per page.

Define the aggregations​

There are a few ways to define the expression for each target column.

  • Use the visual expression builder in the Visual view.
  • Switch to the Code view and write SQL expressions.
  • Ask Copilot to write your expression in either view.

The following table shows how to write expressions for this example.

Target ColumnExpression (Visual)Expression (Code)
page_urlClick Column > page_url.page_url
page_url_countClick Function > count. Then, click Column > page_url inside the visual function.count(page_url)
page_avg_sessionClick Function > avg. Then, click Column > session_duration inside the visual function.avg(session_duration)

Generate data previews​

At this point, you may be curious to know what your data looks like. Generate data previews with the following steps:

  1. Click the play button in the bottom right corner of the canvas.
  2. As the pipeline runs, preview icons should appear as gem outputs.
  3. Click on the Aggregate output to preview the data in the Data Explorer.

Write the data​

The final step in the pipeline captures the output data. Follow these steps to write the data to the SQL warehouse:

  1. From the Source/Target gem category, add a Table gem to your canvas.
  2. Drag the Table gem near the Aggregate gem’s output to auto-connect them.
  3. Open the Table gem configuration.

Unlike the previous step where we configured the table as a seed, we will now set this target as a table.

  1. Select + New Table.
  2. Set the Type and Format to Table, then click Next.
  3. Choose a location for the table based on your SQL connection. You can select an existing table or create a new one.
  4. Click Next, review the table schema, then click Next again.
  5. In the Write Options tab, set Write Mode to Overwrite to replace the table each time the pipeline runs.
  6. Click Next and then Load Data to preview a sample of the target data.

Schedule pipeline runs​

Assume you want to process new web traffic data every day. Create a schedule that runs every morning to integrate and process new data from the previous day.

  1. Expand the Options (ellipses) menu in the project header.
  2. Click Schedule.
  3. Fill in the following parameters.
    • Frequency: Daily
    • Repeat at: 6:00 AM
    • Timezone: Keep the default (the timezone of your current location)
  4. Click Schedule to save the schedule.

Even though schedules are enabled by default, they do not begin to run until the project is published. You don't want to run schedules in production on a draft project! We will save and publish the project in the following sections.

Save your work​

You want to periodically save your work in logical stages. To do so:

  1. In the project header, select Save to Draft.
  2. Give your saved version a description like Aggregate daily web traffic data.
  3. Review the Changes since last saved. You will see the new pipeline and tables you created.
  4. Toggle Show changed files to view the files that will be changed in the backend.
  5. Click Save.

Now, you will be able to view this version of the pipeline in the project version history.

Publish your project​

Let's publish this version of the project. When you publish a project, it becomes available for external use in scheduling, business applications, and packages.

  1. In the project header, select Publish.
  2. Review the Copilot-generated description of the project version.
  3. Notice that the changes to be published include the initial project creation and your saved draft.
  4. Select one or more fabrics to publish the project to. This dictates the fabric(s) that your scheduled pipeline will run on. Prophecy creates a new deployment for each fabric selected.
  5. Click Publish.

As Prophecy publishes your project, a set of logs are generated. Click through each step like fetching fabric info, packaging project, or deploying to fabric to understand the success (or failure) of your project deployment.

See Also

To further your understanding about publication, visit Project publication.

Monitor deployed projects​

At this point, you have built a pipeline, scheduled a periodic pipeline run, and published the related project. This means that your schedule has been activated. To view deployed projects and pipeline schedules:

  1. Open the Monitoring page from the left navigation bar.
  2. Find your deployed project on the Deployed Projects tab.
  3. Find your pipeline on the Scheduled Pipelines tab.
  4. Review the record of your pipeline runs across deployed projects in the Run History tab.
info

To disable a pipeline schedule, you must do so in the schedule settings AND republish the project.

What's next​

To make pipeline development even easier, add connections to browse through your different external data sources. To create more complex pipelines, review the list of gems(/analysts/gems) that are available to you and try them out!
